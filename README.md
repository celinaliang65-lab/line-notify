# 📢 LINE 班群自動通知系統

> 每天早上 **06:30（台灣時間）** 自動讀取 Excel 排程表，依設定日期將提醒訊息推播至指定 LINE 班群。

---

## 📋 系統資訊

| 項目 | 說明 |
|------|------|
| 程式檔案 | `notify.py` |
| 執行環境 | GitHub Actions |
| 排程時間 | 每天 06:30（台灣時間）/ Cron: `30 22 * * *`（UTC） |
| 通知管道 | LINE Messaging API — Flex Message 格式 |
| 資料來源 | `data/LINE班群提醒排程.xlsx`（工作表：提醒排程） |
| 環境變數 | `LINE_CHANNEL_ACCESS_TOKEN`（GitHub Secrets 設定） |

## 📊 Excel 欄位說明

排程表第 1 列為欄位標題，第 2 列為說明列（程式自動跳過），**資料從第 3 列開始填寫**。

| 欄位名稱 | 格式 | 說明 |
|----------|------|------|
| 提醒類別 | 文字 | `作業繳交` / `上台規範` / `公告`（決定 badge 顏色） |
| 班群名稱 | 文字 | 班群顯示名稱，僅供辨識，不影響發送 |
| 目標班群 ID | 文字 | LINE Group ID（一長字串） |
| 提醒日期 | **日期格式** | 設定發送日期（請使用 Excel 日期格式，非文字） |
| 訊息標題 | 文字 | 顯示在 Flex Message 標題列的簡短標題 |
| 提醒內容細節 | 文字 | 完整推播內容，支援換行 |
| 備註 | 文字 | 補充說明，程式不讀取，不影響發送 |
| 已發送 | 程式自動填入 | 發送成功後自動填入 `TRUE`，請勿手動修改 |
| 發送時間 | 程式自動填入 | 發送成功時間，程式自動填入，請勿手動修改 |

## ⚙️ 執行流程

1. **取得 LINE Token** — 從 GitHub Secrets 讀取 `LINE_CHANNEL_ACCESS_TOKEN`
2. **讀取 Excel** — 讀取排程表，跳過第 2 列說明列
3. **日期解析** — 日期欄保留原始型別，確保正確解析
4. **Debug Log** — 印出每筆資料的日期、已發送狀態供排查
5. **篩選資料** — 找出「今天日期」且「已發送為空」的資料
6. **發送訊息** — 依序透過 LINE API 推播 Flex Message 至各班群
7. **回寫狀態** — 發送成功才回寫 `TRUE` 與時間戳記至 Excel
8. **儲存檔案** — 有任一筆成功才執行存檔，全部失敗則不回寫
9. **推送 GitHub** — Workflow 將更新後的 Excel 推回 repo

---

## ⚠️ 使用注意事項

- **提醒日期欄請保持「日期格式」**，不可改成文字格式，否則程式無法正確讀取
- **目標班群 ID 須為正確的 LINE Group ID**（C 開頭長字串），填錯會導致 LINE API 400 錯誤
- **已發送與發送時間欄位由程式自動填入**，請勿手動修改
- 發送失敗的資料不會被回寫，下次執行時會自動重試
- 若需重新發送已成功的資料，需手動清空該筆的已發送與發送時間欄位後，從 GitHub 重新上傳
- 新增班群前請先確認 LINE Bot 已加入該群組，否則無法發送

|------|------|
| Cron 排程 | `30 22 * * *`（UTC）= 台灣時間每天 06:30 |

| 寫回 Excel | 由 workflow 執行 `git commit & push` 推回 repo |
